## 🚀 Быстрый старт для нового владельца

### 1. Получение проекта

```bash
# Вариант A: Клонирование (если настроен удалённый репозиторий)
git clone <repo-url>
cd SABot

# Вариант B: Архив (если передача через файл)
# Распакуйте архив, перейдите в папку SABot
```

### 2. Настройка окружения

```bash
# Создание виртуального окружения
python -m venv .venv

# Активация (Windows PowerShell)
.venv\Scripts\Activate.ps1

# Активация (Windows CMD)
.venv\Scripts\activate.bat

# Активация (Linux/Mac)
source .venv/bin/activate

# Установка зависимостей
pip install -r requirements.txt
```

### 3. Конфигурация

```bash
# Копирование шаблона конфигурации
cp .env.example .env
```

Отредактируйте файл `.env`:

```env
# Получите токен у @BotFather
BOT_TOKEN=your_new_bot_token_here

# Оставьте как есть
DB_NAME=user_roles.db

# ВАЖНО: Ваш Telegram ID (получить у @userinfobot)
INITIAL_ADMIN_ID=123456789
```

### 4. Первый запуск

```bash
python main.py
```

При первом запуске:
1. Бот создаст пустую базу данных
2. Автоматически назначит вас администратором (по `INITIAL_ADMIN_ID`)
3. Запишет в лог: "Добавлен начальный админ (ID: 123456789)"

## 🔐 Как сделать репозиторий приватным

### Вариант 1: GitHub (рекомендуется)

1. Создайте **приватный** репозиторий на GitHub:
   - НЕ ставьте галочку "Initialize this repository with a README"
   - Выберите "Private"

2. Привяжите локальный репозиторий:

```bash
# Добавьте удалённый репозиторий
git remote add origin https://github.com/ВАШ_НИКНЕЙМ/НАЗВАНИЕ_РЕПО.git

# Отправьте код
git push -u origin master
```

3. Настройка доступа (кто может видеть код):
   - Settings → Manage access → Invite a collaborator
   - Или используйте deploy keys для деплоя

### Вариант 2: GitLab (бесплатные приватные репозитории)

```bash
# Создайте проект на gitlab.com (private)
# Затем:
git remote add origin https://gitlab.com/ВАШ_НИКНЕЙМ/sabot.git
git push -u origin master
```

### Вариант 3: Локальный сервер (self-hosted)

Если хотите полный контроль:
- **Gitea** — лёгкий self-hosted GitHub
- **Gogs** — минималистичный вариант

### Вариант 4: Без облака (архив)

Если не хотите использовать git хостинг:

```bash
# Создание архива для передачи
git bundle create sabot.bundle --all

# Распаковка на другом компьютере
git clone sabot.bundle
```

## ⚠️ Важные замечания

### Файлы, которые НЕ попадают в git (и правильно!)

| Файл | Причина |
|------|---------|
| `.env` | Содержит секреты (токен) |
| `*.db` | База данных с пользователями |
| `__pycache__/` | Кэш Python |
| `.venv/` | Виртуальное окружение |
| `*.log` | Логи |

### Что нужно передать отдельно

1. **Текущая база данных** (`user_roles.db`) — если нужно сохранить пользователей
2. **Актуальный `.env`** — но только через защищённый канал!

### Рекомендуемый способ передачи секретов

```
.env файл → Password manager / Secure messenger
БД (если нужна) → Зашифрованный архив
Код → Git (приватный репозиторий)
```

## 🐛 Возможные проблемы при передаче

### Проблема: "У вас нет доступа к боту"

**Причина:** `INITIAL_ADMIN_ID` не задан или неверный ID.

**Решение:**
1. Получите свой ID у @userinfobot
2. Запишите в `.env`
3. Удалите `user_roles.db` (если она пустая)
4. Перезапустите бота

### Проблема: "BOT_TOKEN не найден"

**Причина:** Нет файла `.env` или в нём неверные данные.

**Решение:**
```bash
cp .env.example .env
# Отредактируйте .env
```

### Проблема: "ModuleNotFoundError: No module named 'aiogram'"

**Причина:** Виртуальное окружение не активировано.

**Решение:**
```bash
# Windows PowerShell
.venv\Scripts\Activate.ps1

# Если ошибка политик выполнения, выполните (от администратора):
# Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```
