## 🐛 Найденные проблемы

### BUG-001: Необработанный callback_query "noop" в событиях
**Статус:** ⚠️ Minor  
**Приоритет:** Medium  
**Модуль:** `handlers/events.py` (строка 242-244)

**Описание:**
При создании события с анонсом в группе, создаются inline-кнопки "✅ Иду" / "❌ Не иду" с `callback_data="noop"`. Однако нет обработчика для этого callback, что приведёт к ошибке "Callback query is not answered" и "подвисшим" кнопкам.

**Код:**
```python
rsvp_kb = InlineKeyboardMarkup(inline_keyboard=[
    [
        InlineKeyboardButton(text="✅ Иду", callback_data="noop"),
        InlineKeyboardButton(text="❌ Не иду", callback_data="noop")
    ]
])
```

**Рекомендация:**
Добавить обработчик:
```python
@router.callback_query(F.data == "noop")
async def noop_callback(callback: CallbackQuery):
    await callback.answer("Функция в разработке", show_alert=False)
```

### BUG-002: Нет валидации даты в будущем при редактировании события
**Статус:** ⚠️ Minor  
**Приоритет:** Medium  
**Модуль:** `handlers/events.py` (строка 337-342)

**Описание:**
При создании события проверяется что дата в будущем (`datetime.fromisoformat(dt) <= datetime.now()`), но при редактировании такой проверки нет. Можно случайно изменить событие на дату в прошлом.

**Код создания (есть проверка):**
```python
if datetime.fromisoformat(dt) <= datetime.now():
    await message.answer("❌ Дата должна быть в будущем!")
    return
```

**Код редактирования (нет проверки):**
```python
if len(parts) > 1 and parts[1] != '.':
    try:
        datetime.fromisoformat(parts[1])
        updates['event_datetime'] = parts[1]
    except ValueError:
        ...
```

**Рекомендация:** Добавить проверку на будущее время при редактировании.

### BUG-003: Возможная потеря бан-левела при повторном бане
**Статус:** ⚠️ Info  
**Приоритет:** Low  
**Модуль:** `db_utils.py` (строка 540-542)

**Описание:**
В `apply_ban` используется `INSERT OR REPLACE`, что может перезаписать существующую запись бана вместо создания новой. Хотя логика определения `ban_level` вычисляется до этого, архитектурно это риск.

**Код:**
```python
await db.execute(
    "INSERT OR REPLACE INTO bans (user_id, username, ban_level, banned_until) VALUES (?, ?, ?, ?)",
    (user_id, username, ban_level, banned_until.isoformat())
)
```

**Рекомендация:** Использовать обычный INSERT или проверять существующие баны.

### NOTE-004: Обработка ошибок при отправке в группу
**Модуль:** `handlers/events.py` (строка 256-260)

**Описание:**
```python
try:
    await bot.send_message(**kwargs)
    await message.answer("📢 Анонс размещён в группе!")
except Exception as e:
    logging.error(f"Ошибка отправки в группу: {e}")
    await message.answer("⚠️ Событие сохранено, но не удалось разместить анонс в группе.")
```

**Статус:** Так и задумано, но можно улучшить диагностику.
