### ✅ Ошибка 1: Парсинг даты (ИСПРАВЛЕНО)
**Проблема:** `15.03.2026` возвращал `None`

**Причина:** `transformer` в `parse_date_flexible` не использовался

**Исправление:** Упрощена логика функции:
```python
# Было (не работало):
patterns = [
    (r'^(\d{1,2})\.(\d{1,2})\.(\d{4})$', lambda d, m, y: (d, m, y[2:])),
]
# transformer не использовался!

# Стало:
match = re.match(r'^(\d{1,2})\.(\d{1,2})\.(\d{2,4})$', normalized)
if len(year) == 4:
    year = year[2:]
```
