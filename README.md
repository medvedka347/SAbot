# 🤖 SABot - Бот для комьюнити системного анализа

Telegram-бот для управления учебным комьюнити системного анализа. Модульная архитектура, ролевая модель, распределение материалов по стадиям обучения, управление событиями и система банов для защиты.

## ✨ Возможности

### 👥 Ролевая модель
- **User** — доступ к материалам, событиям, записи на мок
- **Mentor** — расширенный доступ + панель ментора
- **Admin** — полный контроль: материалы, события, роли, баны

### 📚 Материалы по стадиям обучения
- 📖 **Фундаментальная теория** — базовые концепции
- 🔧 **Практическая теория** — применение знаний  
- 📝 **Практические задания** — задачи для отработки
- 🗺️ **Roadmap** — план обучения

### 📅 Управление событиями
- CRUD для вебинаров, митапов, квизов
- Автоматическая публикация анонсов в группу
- Просмотр предстоящих событий

### ⏱️ Запись на мок-интервью
- Календари менторов (интеграция с cal.com)
- Инструкции по подготовке
- Контакты собеседующих

### 🔍 Поиск по материалам
- Команда `/search <запрос>` в личных сообщениях
- Команда `/material <название>` в группах
- Поиск по названию и описанию

### 🛡️ Система безопасности
- Бан после 3 неудачных попыток авторизации
- Прогрессивные наказания: 5 мин → 10 мин → 1 месяц
- Автоматическая очистка истекших банов

## 🚀 Установка и запуск

### 1. Клонирование и зависимости

```bash
git clone https://github.com/medvedka347/SABot.git
cd SABot
pip install -r requirements.txt
```

### 2. Настройка окружения

```bash
cp .env.example .env
```

Отредактируйте `.env`:

```env
BOT_TOKEN=your_bot_token_from_botfather
DB_NAME=user_roles.db
INITIAL_ADMIN_ID=your_telegram_id
```

**Получение Telegram ID:** напишите [@userinfobot](https://t.me/userinfobot)

### 3. Запуск

```bash
python main.py
```

При первом запуске бот автоматически создаст администратора с ID из `INITIAL_ADMIN_ID`.

## 🏗️ Архитектура

### Middleware (AuthMiddleware)

Централизованная проверка авторизации:
- Проверяет пользователя в БД один раз на сообщение
- Кеширует роль в `data["user_role"]` для последующих фильтров
- Блокирует неавторизованных с уведомлением

### Фильтры

```python
HasRole(ROLE_ADMIN)      # Проверка конкретной роли
IsAuthorizedUser         # Проверка наличия в БД (fallback)
```

### FSM States

Каждый модуль имеет изолированные состояния:
- `MaterialStates` — для работы с материалами
- `EventStates` — для работы с событиями  
- `RoleStates` — для управления ролями

**Управление материалами:**
```
⚙️ Админка → 📦 Управление материалами
→ 📖 Просмотреть | ➕ Добавить | ✏️ Редактировать | 🗑️ Удалить | 📊 Статистика
```

```
⚙️ Админка → 📋 Управление событиями

```

**Управление пользователями:**
```
⚙️ Админка → 👥 Управление ролями
→ 📋 Список | ➕ Назначить роль | 🗑️ Удалить пользователя
```

Форматы добавления пользователей:
- `123456789` — только ID
- `@ivan_petrov` — только username
- `123456789 @ivan_petrov` — связать ID и username
- `@ivan, @petr, 123456789` — несколько пользователей

## 🗄️ База данных

### Таблицы

**user_roles** — пользователи
```sql
id | user_id | username | role | created_at
```

**materials** — материалы
```sql
id | stage | title | link | description | created_at
```

**events** — события
```sql
id | event_type | event_datetime | link | announcement | created_at
```

**bans** — блокировки
```sql
id | user_id | username | ban_level | banned_until | created_at
```

**failed_attempts** — неудачные попытки входа
```sql
id | user_id | username | attempt_count | last_attempt
```

## ⚙️ Технические детали

### Стек
- **Python 3.11+**
- **aiogram 3.x** — асинхронный фреймворк
- **aiosqlite** — асинхронный SQLite
- **python-dotenv** — переменные окружения

### Особенности
- **WAL mode** для SQLite — поддержка конкурентного доступа
- **Rate limiting** — защита от спама (10 сек / 20 запросов)
- **FSM** — Finite State Machine для диалогов
- **Модульность** — каждый домен в отдельном файле

### Безопасность
- Валидация ID (0 < ID < 10^10)
- Нормализация username (lowercase, без @)
- Проверка URL (http/https)
- Защита от SQL-инъекций через параметризованные запросы
