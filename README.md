# 🤖 SABot - Учебный бот для системного анализа

Telegram-бот для управления учебным комьюнити системного анализа. Поддерживает ролевую модель, распределение материалов по стадиям обучения, управление событиями и защиту от несанкционированного доступа.

## ✨ Возможности

### 👥 Ролевая модель
- **User** — доступ к материалам и событиям
- **Mentor** — расширенный доступ + админ-панель (частично)
- **Admin** — полный контроль над ботом

### 📚 Материалы по стадиям обучения
- 📖 **Фундаментальная теория** — базовые концепции
- 🔧 **Практическая теория** — применение знаний
- 📝 **Практические задания** — задачи для отработки
- 🗺️ **Roadmap** — план обучения и дополнительная информация

### 📅 Управление событиями
- Добавление вебинаров, митапов, квизов
- Редактирование и удаление событий
- Просмотр предстоящих событий для всех пользователей

### 🛡️ Защита от брутфорса
- Бан после 3 неудачных попыток авторизации
- Прогрессивная система наказаний:
  - Уровень 1: бан на 5 минут
  - Уровень 2: бан на 10 минут
  - Уровень 3: бан на 1 месяц
- Автоматическая очистка истекших банов

### 👤 Гибкое управление пользователями
- Добавление по Telegram ID (`123456789`)
- Добавление по username (`@ivan_petrov`)
- Связывание ID + username в одной записи
- Массовое назначение ролей

## 🚀 Установка и запуск

### 1. Клонирование и зависимости
```bash
git clone <repo-url>
cd SABot
pip install -r requirements.txt
```

### 2. Настройка окружения
Создайте файл `.env` (скопируйте из `.env.example`):
```bash
cp .env.example .env
```

Отредактируйте `.env`:
```env
BOT_TOKEN=your_bot_token_from_botfather
DB_NAME=user_roles.db
INITIAL_ADMIN_ID=your_telegram_id_here
```

**Как получить свой Telegram ID:**
1. Напишите боту [@userinfobot](https://t.me/userinfobot)
2. Он пришлёт ваш ID — скопируйте его в `INITIAL_ADMIN_ID`

### 3. Настройка начального админа
При первом запуске бот автоматически создаст администратора с ID из `INITIAL_ADMIN_ID`.

> ⚠️ **Важно:** Если `INITIAL_ADMIN_ID` не задан, бот запустится, но доступа не будет. Установите ID перед первым запуском!

### 4. Запуск
```bash
python main.py
```

## 🎯 Использование

### Для обычных пользователей
1. Нажмите `/start` — бот проверит вашу роль
2. **📚 Материалы** — выберите нужный раздел (теория, практика, roadmap)
3. **📅 События комьюнити** — просмотр предстоящих мероприятий
4. **⏱️ Записаться на мок** — функция в разработке

### Для администраторов

#### Управление ролями
```
⚙️ Админка → 👥 Управление ролями
```

**Форматы добавления пользователей:**
- Только ID: `123456789`
- Только username: `@ivan_petrov`
- Оба значения: `123456789 @ivan_petrov` (связываются в БД)
- Несколько пользователей: `@ivan, @petr, 123456789`

#### Управление материалами
```
⚙️ Админка → 📦 Управление материалами
```
- **📖 Просмотреть** — список материалов по разделам
- **➕ Добавить** — добавление с указанием стадии
- **✏️ Редактировать** — изменение через inline-кнопки
- **🗑️ Удалить** — удаление материала
- **📊 Статистика** — количество материалов по разделам

#### Управление событиями
```
⚙️ Админка → 📋 Управление событиями
```
- Полный CRUD для событий
- Формат даты: `2024-12-31 18:00:00`

## 🗄️ База данных

### Основные таблицы

**user_roles** — пользователи и роли
```sql
id | user_id | username | role | created_at
```

**materials** — учебные материалы
```sql
id | stage | title | link | description | created_at
```

**events** — события комьюнити
```sql
id | event_type | event_datetime | link | announcement | created_at
```

**bans** — временные блокировки
```sql
id | user_id | username | ban_level | banned_until | created_at
```

**failed_attempts** — счетчик неудачных попыток
```sql
id | user_id | username | attempt_count | last_attempt
```

## ⚙️ Технические детали

### Стек технологий
- **Python 3.11+**
- **aiogram 3.x** — асинхронный фреймворк для Telegram Bot API
- **SQLite** — встроенная БД (не требует установки)
- **python-dotenv** — работа с переменными окружения

### Особенности реализации
- **Контекстные менеджеры** для работы с БД — гарантированное закрытие соединений
- **FSM (Finite State Machine)** — управление многошаговыми диалогами
- **Универсальная авторизация** — поиск пользователя по ID ИЛИ username
- **Автоматические миграции** — обновление схемы БД при изменениях

### Безопасность
- Валидация ID пользователей (0 < ID < 10^10)
- Нормализация username (lower case, без @)
- Скрытие точного времени разблокировки
- Проверка схемы URL для материалов
