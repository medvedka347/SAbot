## 🔴 КРИТИЧЕСКИЕ БАГИ (Agent QA)

### BUG-001: Ошибка логики в отчёте Льва
**Файл:** `handlers/buddy.py:611`  
**Суть:** В отчёте по ментору показывается информация о текущем пользователе (Льве), а не о выбранном менторе.

```python
# НЕПРАВИЛЬНО:
mentor = await get_user_by_id(callback.from_user.id)  # Всегда Лев!

# ПРАВИЛЬНО:
mentor = await get_user_by_id(mentor_id)  # Выбранный ментор
```

**Влияние:** 🔴 Высокое — функция отчётов неработоспособна

---

### BUG-002: Race Condition при редактировании
**Файл:** `handlers/materials.py:271-304`  
**Суть:** Два админа могут одновременно редактировать один материал, и второе сохранение перезапишет первое без предупреждения.

**Сценарий:**
1. Админ А выбирает материал ID=5
2. Админ Б выбирает тот же материал
3. Админ А сохраняет изменения
4. Админ Б сохраняет — перезаписывает изменения А

**Влияние:** 🔴 Высокое — потеря данных

---

### BUG-003: Проблемы с мультиролями
**Файлы:** `handlers/common.py:97-102`, `common.py:130-143`, `common.py:165-173`  
**Суть:** Проверка `role == ROLE_ADMIN` не работает для пользователей с ролью `admin,lion` или `admin,mentor`.

```python
# НЕПРАВИЛЬНО:
role = await get_user_role(user_id)
if role == ROLE_ADMIN:  # False для 'admin,lion'

# ПРАВИЛЬНО:
roles = await get_user_roles(user_id)
if ROLE_ADMIN in roles:
```

**Влияние:** 🔴 Высокое — пользователи с мультиролями теряют доступ к функциям

---

### BUG-004: get_all_mentors не видит мультироли
**Файл:** `db_utils.py:1017-1033`  
**Суть:** Менторы с ролью `mentor,admin` не попадают в список для Льва.

```sql
-- НЕПРАВИЛЬНО:
WHERE role = 'mentor'  -- Точное совпадение

-- ПРАВИЛЬНО:
WHERE role LIKE '%mentor%'  -- Подстрока
```

**Влияние:** 🔴 Высокое — Лев не видит часть менторов

## 🟡 БАГИ СРЕДНЕГО УРОВНЯ

### BUG-005: Нет проверки прав доступа к чужому менти
**Файл:** `handlers/buddy.py:403-426`  
**Суть:** Ментор может просмотреть чужого менти, перебирая ID в callback.

---

### BUG-006: FSM state не сбрасывается корректно
**Файл:** `handlers/materials.py:321-338`  
**Суть:** После удаления материала state сбрасывается, но пользователь остаётся в меню. При нажатии кнопки — "❌ Нет доступа".

---

### BUG-007: Неконсистентность валидации ссылок
**Файлы:** `handlers/materials.py:183-194`, `handlers/events.py:132-142`  
**Суть:** Валидация только по префиксу `http://`, но `utils.py` содержит полноценную `is_valid_url()` которая не используется.

---

### BUG-008: Неправильная навигация назад
**Файл:** `handlers/common.py:191-242`  
**Суть:** Ключ `input_link_evt` в STATE_MAP, но в `back_messages` используется неправильное имя состояния.

---

### BUG-009: Дублирующийся update_data в ролях
**Файл:** `handlers/roles.py:252-254`  
**Суть:** `_prev_state` сначала устанавливается в `input_users`, потом сразу перезаписывается на `selecting_role`. Навигация "Назад" сломана.

---

### BUG-010: Callback кнопки "Иду/Не иду" не работают
**Файл:** `handlers/events.py:242-245`  
**Суть:** Кнопки RSVP имеют callback_data="noop" без обработчика. Пользователь думает что записался, но ничего не происходит.

### 📋 ДЕТАЛЬНЫЙ UX АНАЛИЗ

#### 1. Проблемы пользовательского пути

**Проблема: Разрыв контекста при редактировании**

```python
# Текущий подход (ПЛОХО):
"Отправьте новые данные в формате:\n"
"`название\\n\\nссылка\\n\\nописание`\n\n"
"Используйте '.' для пропуска поля"
```

**Почему это плохо:**
- Пользователь должен помнить текущие значения
- Двойной перенос строки неочевиден
- Точка для пропуска — нестандартно

**Рекомендация:**
```python
# Улучшенный подход:
"Текущее название: {current_title}\n"
"Введите новое название (или отправьте . для пропуска):"
# Пошаговое редактирование каждого поля
```

---

#### 2. Проблемы интерфейса

**Неочевидные эмодзи:**

| Эмодзи | Текущее значение | Проблема | Рекомендация |
|--------|------------------|----------|--------------|
| ⏱️ | Запись на мок | Часы = ожидание | 🎤 Мок-интервью |
| ⚙️ | Админка | Шестерёнка = настройки | 👑 Админ-панель |
| 📦 | Управление материалами | Коробка ≠ материалы | 📚 Материалы (упр.) |
| 🦁 | Панель Льва | Внутренний термин | 👑 Мета-админ |
